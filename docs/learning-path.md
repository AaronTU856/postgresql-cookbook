# Learning Path

The examples are ordered so learners can build confidence step by step. Start with simple reads before moving into relationships, calculations, and PostgreSQL-specific features.

## Recommended Order

1. **Basic queries**
   Learn how to select columns, limit result sets, and read from a single table.

2. **Filtering and sorting**
   Learn how to find specific records and order results for reports or application screens.

3. **Joins**
   Learn how related tables work together. This is essential for backend development and Django work.

4. **Aggregates**
   Learn how to answer reporting questions such as total revenue, order counts, and average prices.

5. **Common table expressions**
   Learn how to break complex queries into readable steps.

6. **Window functions**
   Learn how to rank, compare, and calculate values across rows without losing row-level detail.

7. **Transactions**
   Learn how to keep data consistent when multiple changes must succeed or fail together.

8. **Indexes and performance**
   Learn how PostgreSQL finds data efficiently and how to inspect query plans.

9. **Views, functions, and triggers**
   Learn how to package database logic responsibly.

10. **JSONB and Django examples**
    Learn PostgreSQL features commonly used in modern web applications.

## Suggested Practice Routine

- Read the example once without running it.
- Predict the result.
- Run the query in `psql`.
- Change one condition or column.
- Write down what changed and why.
- Complete the challenge exercise before moving on.
